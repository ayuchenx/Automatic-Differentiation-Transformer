The intuition behind fused operators is to improve computation performance by combing multiple operations into a single kernel. Usually the results from matrix multiplication would be written into memory before being read/accessed again for the next step. However, fused operators are able to reduce the memory usage by removing the need to store and access intermediate results. For example, in our case here, instead of obtaining a result from matrix multiplication and then passing it into a separate layer for layernorm or softmax, we can just execute both processes in the same kernel. This helps reduce the read and write operations and minimizes extra memory access, as data transfer can often be a bottleneck in deep learning. Additionally, it also reduces the number of times we need to launch a kernel. Some future improvements for the operators can be automatically fusing operators together in a computational graph, which would help reduce manual efforts. For my fused operators, the two different operators were just stacked on top of each other. An area of improvement for this could be to make it more efficient as some patterns can arise when the two operators are working together. They could share similar operations and that can be taken advantage of by only doing the calculation once and reusing it.
